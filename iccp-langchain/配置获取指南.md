# ICCP LangChain 环境配置获取指南

本文档详细说明如何获取 `.env.example` 中各项配置的具体值。

## 一、应用基础配置

### 1. APP_NAME
- **说明**: 应用名称，可自定义
- **获取方式**: 直接填写，例如：`ICCP LangChain API`
- **示例**: `APP_NAME=ICCP LangChain API`

### 2. DEBUG
- **说明**: 调试模式开关
- **获取方式**: 
  - 开发环境：`DEBUG=True`
  - 生产环境：`DEBUG=False`
- **示例**: `DEBUG=False`

### 3. VERSION
- **说明**: 应用版本号
- **获取方式**: 根据项目版本填写
- **示例**: `VERSION=1.0.0`

## 二、数据库配置（PostgreSQL）

### 1. DATABASE_URL 和 DATABASE_SYNC_URL

#### 方式一：使用本地PostgreSQL
**步骤：**
1. **安装PostgreSQL**
   - Windows: 下载安装包 https://www.postgresql.org/download/windows/
   - Mac: `brew install postgresql`
   - Linux: `sudo apt-get install postgresql` 或 `sudo yum install postgresql`

2. **启动PostgreSQL服务**
   ```bash
   # Windows (以管理员身份运行)
   net start postgresql-x64-14  # 版本号可能不同
   
   # Mac/Linux
   sudo systemctl start postgresql
   # 或
   brew services start postgresql
   ```

3. **创建数据库和用户**
   ```bash
   # 登录PostgreSQL
   psql -U postgres
   
   # 创建数据库
   CREATE DATABASE iccp;
   
   # 创建用户（如果不存在）
   CREATE USER iccp_user WITH PASSWORD 'your_password';
   
   # 授予权限
   GRANT ALL PRIVILEGES ON DATABASE iccp TO iccp_user;
   ```

4. **配置连接字符串**
   ```
   DATABASE_URL=postgresql+asyncpg://iccp_user:your_password@localhost:5432/iccp
   DATABASE_SYNC_URL=postgresql://iccp_user:your_password@localhost:5432/iccp
   ```

#### 方式二：使用云数据库（推荐生产环境）
**选项A：阿里云RDS PostgreSQL**
1. 登录阿里云控制台
2. 进入 RDS 数据库管理
3. 创建PostgreSQL实例
4. 获取连接信息：
   - 主机地址：`your-instance.rds.aliyuncs.com`
   - 端口：`5432`
   - 数据库名：创建时指定
   - 用户名和密码：创建时设置
5. 配置：
   ```
   DATABASE_URL=postgresql+asyncpg://username:password@your-instance.rds.aliyuncs.com:5432/dbname
   ```

**选项B：Supabase（免费额度）**
1. 访问 https://supabase.com
2. 注册账号并创建项目
3. 在项目设置中找到数据库连接信息
4. 配置：
   ```
   DATABASE_URL=postgresql+asyncpg://postgres:your_password@db.xxx.supabase.co:5432/postgres
   ```

**选项C：Neon（免费PostgreSQL）**
1. 访问 https://neon.tech
2. 注册账号并创建项目
3. 获取连接字符串
4. 配置：
   ```
   DATABASE_URL=postgresql+asyncpg://user:password@ep-xxx.us-east-2.aws.neon.tech/neondb
   ```

## 三、Redis配置

### REDIS_URL

#### 方式一：本地Redis
**步骤：**
1. **安装Redis**
   - Windows: 下载 https://github.com/microsoftarchive/redis/releases 或使用WSL
   - Mac: `brew install redis`
   - Linux: `sudo apt-get install redis-server`

2. **启动Redis**
   ```bash
   # Windows
   redis-server
   
   # Mac/Linux
   redis-server
   # 或作为服务
   sudo systemctl start redis
   ```

3. **配置**
   ```
   REDIS_URL=redis://localhost:6379/0
   ```

#### 方式二：云Redis服务
**选项A：阿里云Redis**
1. 登录阿里云控制台
2. 创建Redis实例
3. 获取连接地址和端口
4. 配置：
   ```
   REDIS_URL=redis://:password@your-instance.redis.rds.aliyuncs.com:6379/0
   ```

**选项B：Upstash（免费额度）**
1. 访问 https://upstash.com
2. 创建Redis数据库
3. 获取连接URL
4. 配置：
   ```
   REDIS_URL=redis://default:password@xxx.upstash.io:6379
   ```

**选项C：Redis Cloud**
1. 访问 https://redis.com/try-free/
2. 注册并创建数据库
3. 获取连接信息
4. 配置连接字符串

## 四、JWT配置

### 1. SECRET_KEY
**步骤：**
1. **生成密钥（推荐使用Python）**
   ```python
   import secrets
   print(secrets.token_urlsafe(32))
   ```
   或使用OpenSSL：
   ```bash
   openssl rand -hex 32
   ```

2. **配置**
   ```
   SECRET_KEY=生成的密钥字符串
   ```
   **⚠️ 重要**: 生产环境必须使用强随机密钥，不要使用示例值！

### 2. ALGORITHM
- **说明**: JWT算法，通常使用HS256
- **配置**: `ALGORITHM=HS256`

### 3. ACCESS_TOKEN_EXPIRE_MINUTES
- **说明**: Token过期时间（分钟）
- **配置**: `ACCESS_TOKEN_EXPIRE_MINUTES=30`（可根据需求调整）

## 五、LLM配置

### 1. OPENAI_API_KEY

#### 选项A：使用OpenAI官方API
**步骤：**
1. 访问 https://platform.openai.com
2. 注册/登录账号
3. 进入 API Keys 页面：https://platform.openai.com/api-keys
4. 点击 "Create new secret key"
5. 复制生成的API密钥
6. 配置：
   ```
   OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   OPENAI_MODEL=gpt-4
   LLM_BASE_URL=https://api.openai.com/v1
   ```

#### 选项B：使用阿里云DashScope（通义千问）
**步骤：**
1. 访问 https://dashscope.aliyun.com
2. 注册/登录阿里云账号
3. 开通DashScope服务
4. 进入API-KEY管理：https://dashscope.console.aliyun.com/apiKey
5. 创建API-KEY
6. 配置：
   ```
   OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   OPENAI_MODEL=qwen-turbo  # 或 qwen-plus, qwen-max
   LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
   ```

#### 选项C：使用其他兼容OpenAI的API
- **DeepSeek**: https://platform.deepseek.com
- **Moonshot**: https://platform.moonshot.cn
- **智谱AI**: https://open.bigmodel.cn
- **零一万物**: https://platform.01.ai

配置格式类似，只需修改 `LLM_BASE_URL` 和 `OPENAI_MODEL`

### 2. LLM_TIMEOUT
- **说明**: API请求超时时间（秒）
- **配置**: `LLM_TIMEOUT=60`（可根据网络情况调整）

### 3. LLM_TEMPERATURE
- **说明**: 模型温度参数（0-1），控制输出随机性
- **配置**: `LLM_TEMPERATURE=0.7`（0.7是常用值，可根据需求调整）

## 七、Milvus配置（向量数据库）

### MILVUS_HOST 和 MILVUS_PORT

#### 方式一：本地Milvus
**步骤：**
1. **使用Docker安装（推荐）**
   ```bash
   docker pull milvusdb/milvus:latest
   docker run -d --name milvus -p 19530:19530 -p 9091:9091 milvusdb/milvus:latest
   ```

2. **配置**
   ```
   MILVUS_HOST=localhost
   MILVUS_PORT=19530
   ```

#### 方式二：Milvus Cloud（云服务）
1. 访问 https://cloud.zilliz.com
2. 注册账号并创建集群
3. 获取连接信息
4. 配置：
   ```
   MILVUS_HOST=your-cluster.milvus.io
   MILVUS_PORT=19530
   ```

**⚠️ 注意**: 如果不需要向量搜索功能，可以暂时不配置。

## 八、CORS配置

### CORS_ORIGINS
**步骤：**
1. 确定前端应用的访问地址
2. 配置允许的源地址列表
3. 配置：
   ```
   CORS_ORIGINS=["http://localhost:3000","http://localhost:5173","https://yourdomain.com"]
   ```
   - `localhost:3000`: 通常为Next.js默认端口
   - `localhost:5173`: 通常为Vite默认端口
   - 生产环境添加实际域名

## 九、Celery配置（异步任务）

### CELERY_BROKER_URL 和 CELERY_BACKEND_URL
**说明**: Celery使用Redis作为消息代理和结果后端

**配置：**
```
CELERY_BROKER_URL=redis://localhost:6379/1
CELERY_BACKEND_URL=redis://localhost:6379/2
```

**注意**: 
- 使用不同的Redis数据库编号（1和2）避免冲突
- 如果使用云Redis，配置格式：
  ```
  CELERY_BROKER_URL=redis://:password@your-redis-host:6379/1
  CELERY_BACKEND_URL=redis://:password@your-redis-host:6379/2
  ```

## 配置验证

配置完成后，可以通过以下方式验证：

1. **检查环境变量加载**
   ```python
   import os
   from dotenv import load_dotenv
   load_dotenv()
   print(os.getenv('OPENAI_API_KEY'))
   ```

2. **测试数据库连接**
   ```python
   import asyncpg
   async def test_db():
       conn = await asyncpg.connect(os.getenv('DATABASE_URL'))
       print("数据库连接成功")
   ```

3. **测试Redis连接**
   ```python
   import redis
   r = redis.from_url(os.getenv('REDIS_URL'))
   r.ping()  # 应该返回True
   ```

## 常见问题

### Q: 开发环境不想安装所有服务怎么办？
A: 可以使用Docker Compose一键启动所有服务，或使用云服务替代。

### Q: 哪些配置是必须的？
A: 最小配置需要：`SECRET_KEY`、`OPENAI_API_KEY`、`DATABASE_URL`、`REDIS_URL`。

### Q: 如何生成强密码？
A: 使用Python: `import secrets; print(secrets.token_urlsafe(32))`

### Q: 生产环境配置注意事项？
A: 
- 使用强随机密钥
- 不要将`.env`文件提交到Git
- 使用环境变量或密钥管理服务
- 启用HTTPS
- 配置防火墙规则
